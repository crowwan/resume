## 작성 원칙

### 문장 패턴 (우선순위)

1. **[동사] + [대상] + [방법] + [결과]**
   - "레거시 시스템을 마이크로서비스로 전환하여 배포 주기를 2주→1일로 단축"

2. **[문제 상황]에서 [해결책]으로 [성과 달성]**
   - "도메인 33개 분산 관리 환경에서 통합 도메인 전략으로 SSL 97% 감소"

3. **[역할]로서 [범위]의 [결과물] 담당**
   - "프론트엔드 전담 개발자로서 SaaS 구독 관리 시스템 전체 설계 및 구현"

### 핵심 규칙

- **정량적 성과 필수**: 수치(%, 개수, 시간)가 없는 성과는 작성하지 않음
- **STAR 기법**: Situation → Task → Action → Result
- **동사로 시작**: "설계", "구현", "주도", "구축", "최적화", "리팩토링"
- **기술적 깊이**: 단순 나열이 아닌 WHY/HOW 포함
- **임팩트 중심**: "~했다" 대신 "~하여 ~를 달성했다"

### 피해야 할 표현

- "~에 참여했습니다" → "~를 전담/주도했습니다"
- "다양한 기술을 사용했습니다" → 구체적 기술과 사용 맥락 명시
- "팀과 협업했습니다" → 구체적 기여 범위와 결과 명시
